Gestion des utilisateurs

L'interface propose un panneau de configuration qui permet de gérer un utilisateur d'un site. Ce panneau traite UNIQUEMENT les utilisateurs du site en cours de navigation. La première page montre les utilisateur sélectionné par critère. Les critères sont : Groupe, Etat (actif/supprimé). L'utilisateur peut filtrer avec une expression contenu dans les identifiants. Ex: « brad » permettra de faire apparaître « BradAnjelina ». Pour accéder au détail de l'utilisateur, cliquez sur la ligne choisie. Le page des détails permet de manipuler les informations de l'utilisateur. L'utilisateur pourra rajouter un commentaire que seul les administrateurs du site verront. Finalement il pourra modifier les droit d'accès du site et l'appartenance à un groupe.

Gestion du site

L'interface propose un panneau de configuration qui permet de gérer le site en cours. Plusieurs onglets sont disponibles.

Gestion des journaux

La gestion des journaux est un moyen de repérer facilement les erreurs ou les problèmes de manipulation. Nombre d'opérations sont relatées dans les journaux d'événement. La plupart du temps ils n'ont pas d'utilité pour l'administrateur moyen, mais se révèle très utiles pour le développeur PHP qui souhaite ajouter des fonctionnalités ou créer de nouvelles fonctionnalités a partir de l'existant. L'interface de gestion permet de sélectionner le type d'évènement a visualiser et permet de vider la tables des journaux. Plusieurs fonctions sont proposées de manière a permettre de ne pas faire table rase des journaux et de garder les derniers évènements.

Gestion des modules

La gestion des modules propose un tableau synthétique qui permet de visualiser rapidement l'état des modules. En cliquant sur le module souhaité on obtient la fiche de celui ci. Dans la fiche du module l'utilisateur peut modifier a souhait les informations du module concernant les droit d'accès, le type de décoration et si il y a décoration, ainsi que le fichier appelé. Le fichier étant sensé exister sur le site et être un script PHP (il est appelé par un 'include').
